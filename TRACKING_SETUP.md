# Windows/macOS 활동 정보 수집 설정

이 기능은 본인 기기에서 명시적으로 실행하거나 확장을 설치했을 때만 동작한다.
창 제목과 URL에는 문서명, 검색어, 인증 토큰 등 민감정보가 포함될 수 있으므로
다른 사용자의 기기에서는 반드시 사전 동의를 받아야 한다.

## 전체 데이터 흐름

1. `os-tracker/tracker.js`가 OS 입력과 활성 창 변경을 감지한다.
2. 브라우저 확장의 `background.js`가 활성 탭 변경을 감지한다.
3. 두 수집기는 이벤트 배열을 `POST http://localhost:4000/api/metrics`로 보낸다.
4. `backend/src/routes/metrics.js`가 형식을 확인하고 서버 콘솔에 출력한다.
5. 현재 구현은 DB에 영구 저장하지 않는다.

## 1. 백엔드 실행

```powershell
cd backend
npm.cmd install
npm.cmd run dev
```

`http://localhost:4000/api/health`가 `{ "ok": true }`를 반환하면 준비된 상태다.

## 2. OS 활성 창 수집기

```powershell
cd os-tracker
npm.cmd install
npm.cmd start
```

2초마다 활성 창을 확인하고 창이 바뀌면 다음 정보를 버퍼에 넣는다.

- 운영체제
- 앱 이름
- 창 제목
- PID와 실행 파일 경로
- macOS bundle ID
- 창 ID와 프로세스 메모리 사용량

버퍼는 5초마다 백엔드로 전송된다. 전송이 실패하면 다음 전송을 위해 메모리에
다시 넣는다.

### Windows 권한

일반 창 정보는 보통 추가 권한 없이 읽을 수 있다. 관리자 권한으로 실행 중인
프로그램의 입력까지 감지하려면 수집기도 관리자 권한이 필요할 수 있다.

### macOS 권한

- 키보드/마우스 훅: `개인정보 보호 및 보안 → 손쉬운 사용`
- 다른 앱의 창 제목: `개인정보 보호 및 보안 → 화면 및 시스템 오디오 녹음`

권한을 켠 뒤 터미널 또는 수집기를 완전히 재시작해야 할 수 있다. 브라우저 URL은
OS API 결과에 의존하지 않고 아래 확장에서 수집한다.

## 3. Chrome/Edge 확장 설치

1. 백엔드를 먼저 실행한다.
2. Chrome은 `chrome://extensions`, Edge는 `edge://extensions`를 연다.
3. `개발자 모드`를 켠다.
4. `압축해제된 확장 프로그램을 로드`를 누른다.
5. 이 저장소의 `browser-extension` 폴더를 선택한다.

확장은 다음 순간에 활성 탭 정보를 전송한다.

- 브라우저 시작
- 탭 전환
- 탭 URL 또는 제목 변경
- 다른 브라우저 창으로 포커스 이동

수집 항목은 URL, 탭 제목, 탭/창 ID, 발생 시각이다. 시크릿 창은 기본적으로
수집하지 않는다. 백엔드가 꺼져 있으면 최대 500개까지 확장 로컬 저장소에
보관하고 다음 이벤트 때 다시 전송한다.

## 권한 설명

- `tabs`: 활성 탭의 URL과 제목을 읽는다.
- `storage`: 백엔드 전송 실패 시 이벤트를 확장 내부에 임시 보관한다.
- `http://localhost:4000/*`: 확장 서비스 워커가 로컬 백엔드에 요청한다.

확장은 웹페이지 내용을 읽거나 스크립트를 삽입하지 않는다.
